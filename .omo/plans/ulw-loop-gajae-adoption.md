**What you'll get:** The ulw-loop CLI gains the three capabilities worth taking from gajae-code: completing a goal now automatically starts and announces the next one (one call instead of two), several plan-steering changes can be applied in one atomic batch, and groups of related goals can be declared a "validation batch" that forces a proper review gate when the group finishes.

## Scope
### Must have
- `omo ulw-loop checkpoint --status complete` auto-starts the next eligible goal in the SAME invocation and prints the next-goal instruction (or the run-complete / blocked handoff); `--no-advance` restores the old two-call behavior. `complete-goals` keeps working unchanged as the manual fallback.
- `omo ulw-loop steer --proposals-json <json-array-or-path>` applies N steering proposals atomically (all-or-nothing) under one mutation lock, with per-proposal invariant validation, idempotency dedup, and per-proposal ledger audit entries. Mutually exclusive with `--kind`.
- `omo ulw-loop create-goals --validation-batch-json <json>` defines validation batches `{batchId, memberIds, finalGoalId}` stored as optional `plan.validationBatches`; the batch-final member's complete checkpoint requires a quality gate whose `criteriaCoverage` spans ALL member criteria and requires all other members complete; run-final completion requires every batch closed; steering supersede/split swaps replacement ids into batch membership.
