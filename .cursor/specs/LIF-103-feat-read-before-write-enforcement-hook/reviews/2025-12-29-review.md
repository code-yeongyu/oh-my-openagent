### Minor (2)

#### M1: Path exemption uses relative path matching
**Location**: index.ts:22-38
**Observation**: The `isPathExempt` function converts to relative path for matching, which is correct for project-relative patterns. However, if the file is outside the project root, the path won't be properly relativized.
**Recommendation**: This is edge case behavior documented in spec as acceptable. No action required.

#### M2: Log verbosity in production
**Location**: registry.ts:80, index.ts:87, 96, 102, 179, 191
**Observation**: Many log statements fire for normal operations (recording reads, path exemptions, etc.)
**Recommendation**: Consider adding a debug/verbose config option to reduce log noise in production. Low priority.

### Suggestions (3)

#### S1: Consider case-insensitive tool name matching
**Location**: index.ts:158
**Observation**: Tool names are lowercased for comparison, which is good. However, exempt_tools in config are not lowercased.
**Recommendation**: Lowercase both sides or document that exempt_tools must be exact case.

#### S2: Add metrics/stats endpoint
**Location**: registry.ts:116-129
**Observation**: `getStats()` is implemented but not exposed via any tool or hook.
**Recommendation**: Consider adding debug endpoint or periodic stats logging for observability.

#### S3: Consider caching picomatch patterns
**Location**: index.ts:32-35
**Observation**: picomatch is called for each path check. For hot paths with many exemptions, caching compiled matchers could improve performance.
**Recommendation**: Low priority - current performance is acceptable.
