### 🟡 Minor (4)

#### m1: Missing Regex Error Handling in memory_edit

**File**: `src/tools/memory/tools.ts`
**Lines**: 116-118

**Issue**: User-provided regex pattern is compiled without validation. Invalid regex will throw uncaught error.

```typescript
if (args.mode === "regex") {
  const regex = new RegExp(args.needle, "g")  // Can throw SyntaxError
  newContent = content.replace(regex, args.replacement)
}
```

**Impact**: Tool returns generic error message instead of helpful "Invalid regex pattern" message.

**Fix**: Wrap RegExp construction in try/catch with specific error message.

```typescript
if (args.mode === "regex") {
  let regex: RegExp
  try {
    regex = new RegExp(args.needle, "g")
  } catch (e) {
    return JSON.stringify({ success: false, error: `Invalid regex pattern: ${e.message}` }, null, 2)
  }
  newContent = content.replace(regex, args.replacement)
}
```

#### m2: AWS Secret Key Pattern Too Broad

**File**: `src/features/context-learning/secret-redactor.ts`
**Line**: 13

**Issue**: Pattern `[A-Za-z0-9/+=]{40}` matches any 40-char base64 string, causing false positives.

```typescript
{ pattern: /[A-Za-z0-9/+=]{40}(?=\s|$)/g, type: "aws_secret_key" },
```

**Impact**: May redact legitimate content like hashes, UUIDs, or encoded data.

**Fix**: Consider more specific pattern or lower priority. AWS secret keys typically follow specific patterns. Alternatively, add context requirement (e.g., must appear near "aws" or "secret").

---

#### m3: Daily Budget Not Persisted Across Sessions

**File**: `src/hooks/meta-learning-extractor/index.ts`
**Lines**: 36-40, 51-57

**Issue**: `dailySpendUsd` is stored in memory and resets when plugin reloads.

```typescript
function createHookState(): HookState {
  return {
    sessions: new Map(),
    dailySpendUsd: 0,  // Resets on plugin reload
    lastResetDate: new Date().toISOString().split("T")[0],
  }
}
```

**Impact**: Daily budget limit can be bypassed by restarting OpenCode.

**Fix**: For MVP, document this limitation. For future: persist to file or use session storage.

---

#### m4: Memory Tool Names Don't Match Spec

**File**: `src/tools/memory/constants.ts` vs Spec

**Issue**: Spec says tools should be named `omo_write_memory`, `omo_read_memory`, etc. Implementation uses `memory_write`, `memory_read`, etc.

```typescript
// Spec (FR-1):
| omo_write_memory | write_memory* | Write content to memory file |

// Implementation:
WRITE: "memory_write",
```

**Impact**: Minor deviation from spec. Current naming is actually cleaner.

**Fix**: Either update spec to match implementation (recommended) or rename tools. Current naming is more intuitive.
