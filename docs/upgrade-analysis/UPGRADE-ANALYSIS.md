
**核心概念:**

1. **开发上下文模式 (Development Mode)**
   - 核心理念: 先实现后解释、实用主义优先、测试驱动(TDD)
   - 工作流: 编写代码 → 运行测试 → 重构 → 保持提交原子性
   - 优先级: 功能可用性 > 逻辑正确性 > 代码整洁度

2. **代码审查上下文模式 (Review Mode)**
   - 核心理念: 全盘通读、深度评估、安全与质量并重
   - 工作流: 通读变更 → 按严重性排序 → 提供修复示例 → 安全专项分析
   - 审查标准: 逻辑错误、边界条件、OWASP 安全风险、性能瓶颈

3. **研究探索上下文模式 (Research Mode)**
   - 核心理念: 先理解后行动、多问澄清问题、边研究边记录
   - 工作流: 定义问题 → 探索代码/文档 → 形成假设 → 证据验证 → 总结发现
   - 技术增强: 利用"迭代检索"解决子代理上下文不足
