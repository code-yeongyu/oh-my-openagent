### 端到端流程图 - 现有流程 (Current Flow)

> 以下是**现有流程图**，描述了用户直接从 OpenCode 首页切换到 Prometheus agent 时的实际工作流程。
> 
> **关键差异**: 此流程绕过了 Sisyphus 编排器，导致 skill 注入和 defaultSkills 合并不生效。

> **图例说明**:
>
> - `───▶` 表示步骤之间的衔接
> - `【链接机制】` 说明前一步骤如何触发后一步骤
> - `❌` 表示被绕过或未生效的环节
> - `⚠️` 表示依赖 LLM 遵守提示词，无代码强制

> **⚠️ 已识别的问题** (2026-01-22):
>
> | 问题 | 根因 | 影响 |
> |------|------|------|
> | Skill 未注入 | 直接切换 agent 绕过 delegate_task | defaultSkills 配置不生效 |
> | Tiki-Taka 无代码强制 | 完全依赖 LLM 判断 | 可能跳过快速澄清流程 |
> | 执行模式无自动选择 | 依赖 LLM 遵守提示词 | 可能不调用 executing-plans skill |
> | Phase 强制不阻塞 | planning-flow-guide 只警告 | 可能在执行阶段调用规划 agents |

## 目标流程 vs 现有流程 对比

| 阶段 | 目标流程 | 现有流程 | 差距 |
|------|----------|----------|------|
| **入口** | Sisyphus 编排 | 直接切换 Prometheus | 绕过编排系统 |
| **PHASE 0** | 意图门控 + Skill 检查 | ❌ 被绕过 | 无意图分类 |
| **Skill 注入** | delegate_task 自动合并 | ❌ 不生效 | defaultSkills 失效 |
| **Tiki-Taka** | 快速澄清流程 | ⚠️ LLM 自行判断 | 可能跳过 |
| **执行模式** | 代码自动选择 skill | ⚠️ LLM 自行判断 | 可能不调用 skill |
| **完成验证** | skill 指导验证 | ⚠️ 可能跳过 | 不完整验证 |

## 建议修复方案

### 方案 A: 创建 agent-skill-injector hook

当用户直接切换到任何 agent 时，自动加载该 agent 的 defaultSkills。

### 方案 B: 在 Prometheus 提示词中添加 skill 自加载

```typescript
// 在 Prometheus 启动时自动调用
skill("creating-changes")
```

### 方案 C: 强制用户通过 Sisyphus 进入

在 Prometheus 启动时检测是否通过 delegate_task 调用，如果不是则警告用户。
