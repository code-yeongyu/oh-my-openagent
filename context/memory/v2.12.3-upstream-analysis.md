## Executive Summary

Version 2.12.3 introduces **critical OpenCode v1.1.1 compatibility fixes** through a complete permission system migration. The release addresses breaking changes in OpenCode's plugin API where the `tools` configuration was deprecated and merged into a new `permission` system.

### Key Changes
1. **Runtime version detection** - Detects installed OpenCode version via CLI
2. **Automatic permission migration** - Converts between old/new formats at runtime
3. **All agents updated** - 6 builtin agents now use version-aware restrictions
4. **User config migration** - Automatically migrates user/project/plugin agent configs

## Breaking Change Context

### OpenCode v1.1.1 Permission System

**Old Format (pre-1.1.1)**:
```typescript
{
  tools: { bash: true, write: false }
}
```

**New Format (v1.1.1+)**:
```typescript
{
  permission: { 
    bash: "allow",    // Valid SDK keys only
    edit: "deny"      // "write" → "edit"
  }
}
```

**SDK Restrictions**:
- `permission` only accepts: `edit`, `bash`, `webfetch`, `doom_loop`, `external_directory`
- Tool-level restrictions (e.g., `background_task`) must use `tools` config
- Cannot mix `tools` and `permission` for same capability

## Implementation Details

### 1. Version Detection (`src/shared/opencode-version.ts`)

**Evidence** ([permalink](https://github.com/code-yeongyu/oh-my-opencode/blob/b80b37305e0e1c4e5e1f3b6c850a357657e0cd7/src/shared/opencode-version.ts#L35-L54)):

```typescript
export function getOpenCodeVersion(): string | null {
  if (cachedVersion !== NOT_CACHED) {
    return cachedVersion
  }

  try {
    const result = execSync("opencode --version", {
      encoding: "utf-8",
      timeout: 5000,
      stdio: ["pipe", "pipe", "pipe"],
    }).trim()

    const versionMatch = result.match(/(\d+\.\d+\.\d+(?:-[\w.]+)?)/)
    cachedVersion = versionMatch?.[1] ?? null
    return cachedVersion
  } catch {
    cachedVersion = null
    return null
  }
}
```

**How it works**:
- Executes `opencode --version` subprocess
- Parses version string with regex
- Caches result to avoid repeated calls
- Returns `null` on failure (assumes newer version)

**Version comparison**:
```typescript
export function supportsNewPermissionSystem(): boolean {
  const version = getOpenCodeVersion()
  if (!version) return true  // Assume newer if unknown
  return isVersionGte(version, "1.1.1")
}
```

### 2. Permission Compatibility Layer (`src/shared/permission-compat.ts`)

**Evidence** ([permalink](https://github.com/code-yeongyu/oh-my-opencode/blob/b80b37305e0e1c4e5e1f3b6c850a357657e0cd7/src/shared/permission-compat.ts#L15-L29)):

```typescript
export function createAgentToolRestrictions(
  denyTools: string[]
): VersionAwareRestrictions {
  if (supportsNewPermissionSystem()) {
    return {
      permission: Object.fromEntries(
        denyTools.map((tool) => [tool, "deny" as const])
      ),
    }
  }

  return {
    tools: Object.fromEntries(denyTools.map((tool) => [tool, false])),
  }
}
```

**Migration helpers**:
- `migrateToolsToPermission()` - Converts `{ tool: false }` → `{ tool: "deny" }`
- `migratePermissionToTools()` - Converts `{ tool: "deny" }` → `{ tool: false }`
- `migrateAgentConfig()` - Applies migration to full agent config objects

## Technical Architecture

### Permission Migration Flow

```
┌─────────────────────────────────────────────────────────────┐
│ Plugin Initialization                                        │
└─────────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────────┐
│ opencode-version.ts                                          │
│ • execSync("opencode --version")                            │
│ • Parse version string                                       │
│ • Cache result                                               │
│ • supportsNewPermissionSystem() → boolean                   │
└─────────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────────┐
│ Builtin Agent Creation                                       │
│ • createAgentToolRestrictions(["write", "edit"])            │
│ • Returns { permission: {...} } OR { tools: {...} }         │
└─────────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────────┐
│ User Config Migration (config-handler.ts)                    │
│ • Load user agents from JSON                                 │
│ • migrateAgentConfig() for each agent                       │
│ • Convert tools ↔ permission based on version               │
└─────────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────────┐
│ Final Agent Registration                                     │
│ • All agents have correct format for installed version      │
│ • v1.1.1+: permission format                                │
│ • Older: tools format                                        │
└─────────────────────────────────────────────────────────────┘
```

### Version Detection Strategy

**Assumption on failure**: If version detection fails, assume **newer version** (v1.1.1+)

**Rationale**:
- Safer to use new format (forward compatible)
- Older versions will error clearly if wrong format used
- Users on cutting-edge likely have latest OpenCode
