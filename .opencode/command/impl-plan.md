# Implementation Plan

## Overview

Create a full implementation plan considering all variables and possible edge cases. The plan should be detailed, step by step, explaining from an engineer mindset exactly what needs to be done.

## User Input

```text
$ARGUMENTS
```

## Steps

1. **Understand the approach**
   - Analyze the proposed approach or feature
   - Identify all variables and dependencies
   - Note any constraints or requirements

2. **Break down into steps**
   - Create detailed, sequential steps
   - Each step should be a single, actionable task
   - Include all steps from start to finish

3. **Consider edge cases**
   - Identify potential failure points
   - Plan for error handling
   - Consider boundary conditions

4. **Apply first-principles thinking**
   - Question assumptions
   - Evaluate decisions for highest odds of success
   - Use chain-of-thought reasoning

5. **Document the plan**
   - Write clear, instructional steps
   - Include technical details without actual code
   - Note dependencies between steps

6. **Review and validate**
   - Ensure completeness
   - Verify logical flow
   - Check for missing steps

## Plan Structure

### Overview
- What is being implemented
- Why this approach was chosen
- Key success criteria

### Prerequisites
- Required setup or configuration
- Dependencies to install
- Environment requirements

### Implementation Steps
1. Step 1: [Description]
   - Details
   - Considerations
   - Expected outcome

2. Step 2: [Description]
   - Details
   - Considerations
   - Expected outcome

### Edge Cases
- Edge case 1: How to handle
- Edge case 2: How to handle

### Validation
- How to verify implementation
- Test scenarios
- Success criteria
