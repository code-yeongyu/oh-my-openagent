# Add Documentation

## Overview

Add comprehensive documentation for the current code or feature. Include overview, API documentation, implementation details, and examples.

## User Input

```text
$ARGUMENTS
```

## Steps

1. **Analyze documentation needs**
   - Understand what the code/feature does
   - Identify the target audience
   - Determine documentation scope

2. **Create overview documentation**
   - What this code/feature does
   - Why it exists and its purpose
   - Key concepts and terminology

3. **Document API/Interface**
   - Function/method signatures
   - Parameters and return values
   - Example usage with code snippets
   - Error handling and edge cases

4. **Add implementation details**
   - Architecture overview
   - Important design decisions
   - Dependencies and integrations

5. **Provide examples**
   - Common use cases with full examples
   - Best practices and patterns
   - Common pitfalls to avoid

6. **Format and place documentation**
   - Format according to project's documentation standards
   - Place in appropriate location (README, docs folder, or inline comments)
   - Update navigation/index if needed

7. **Call Documentation Master** (for comprehensive docs):
   - Read `.opencode/agent/documentation-master.md`
   - Follow documentation standards
   - Ensure consistency with existing docs

8. **Call Historian** (GOVERNANCE):
   - Read `.opencode/agent/historian.md`
   - Create changelog entry for documentation work
   - Include: documentation type, files created/updated

## Documentation Checklist

- [ ] Overview complete
- [ ] API documented
- [ ] Examples provided
- [ ] Formatting consistent
- [ ] Placed in correct location
- [ ] Navigation updated
