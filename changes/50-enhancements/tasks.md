## ✅ 每个任务的工作流

```
1. 写测试 (RED)
   └─ bun test xxx → 失败 ✓

2. 写代码 (GREEN)  
   └─ bun test xxx → 通过 ✓

3. 优化 (REFACTOR)
   └─ bun test xxx → 仍通过 ✓

4. 提交
   └─ git commit -m "feat(xxx): 描述"
```
