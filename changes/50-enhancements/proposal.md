# Proposal: 43项功能增强

## Problem Statement

oh-my-opencode 项目与 everything-claude-code 对比后发现 9 个核心系统存在 **43 处**可优化点。

> **已移除项目见**: `changes/50-enhancements/removed/archive.md`

---

## 一、MCP 集成系统问题 (4项)

| ID | 问题 | 当前状态 | 影响 |
|:--:|:-----|:---------|:-----|
| **M2** | MCP 配置硬编码 | 写死在 `allBuiltinMcps` | 用户难以自定义，需修改源码 |
| **M3** | 无工具数量限制 | 可加载无限 MCP 工具 | 超过 80 个工具会导致性能问题 |
| **M5** | MCP 无后置检查 | 修改代码后无自动验证 | 可能引入错误而不自知 |
| **M6** | 无健康检查机制 | Remote MCP 不可用时无降级 | 整个系统可能卡住 |

---

## 二、技能系统问题 (4项)

| ID | 问题 | 当前状态 | 影响 |
|:--:|:-----|:---------|:-----|
| **S1** | 语言模式技能格式不统一 | `backend-pattern-*` 系列偏描述性 | 缺少 Iron Law / Do vs Don't 表格 |
| **S2** | Hooks 硬编码在 TS 中 | 无法通过 SKILL.md 配置 | 扩展技能需要修改代码 |
| **S3** | 依赖 AI 主动调用技能 | 需要 AI 知道调用 `skill()` | 经常忘记加载合适的技能 |
| **S4** | 无项目风格探测 | 技能指令是静态的 | 无法适应不同项目的约定 |

---

## 三、上下文管理问题 (5项)

| ID | 问题 | 当前状态 | 影响 |
|:--:|:-----|:---------|:-----|
| **C1** | 无意图模式区分 | 所有任务用相同指令 | 审查任务收到开发指令，效率低 |
| **C2** | 意图未传递到压缩器 | `metis.ts` 分类意图但未传递给 `compaction-context-injector` | 压缩时无法按意图调整权重 |
| **C3** | 被动压缩策略 | 等窗口溢出才压缩 | 丢失重要上下文 |
| **C4** | failure-counter 状态未注入摘要 | `failure-counter` 追踪失败但未传递给压缩 | 重复尝试已失败的方案 |
| **C5** | plan-attention-refresher 内容有限 | 仅刷新 tasks.md，无行为准则 | 长对话 AI Slop 问题严重 |

---

## 四、规则系统问题 (9项)

| ID | 问题 | 当前状态 | 影响 |
|:--:|:-----|:---------|:-----|
| **R1** | 规则不区分角色 | 所有 Agent 收相同规则 | Oracle 收到不相关的开发规则 |
| **R2** | 规则不区分阶段 | 全程注入所有规则 | 上下文冗余，浪费 Token |
| **R3** | TDD 验证是假的 | `hasFailingTest` 硬编码 false | TDD 守卫形同虚设 |
| **R4** | 测试覆盖检测粗糙 | 仅检查同名测试文件 | 无法验证测试是否真正覆盖目标函数 |
| **R5** | 无测试隔离性检查 | 不检测数据库/网络操作 | 测试不可靠 |
| **R6** | 无高风险操作审计 | 删除/执行无额外检查 | 危险操作无预警 |
| **R7** | 无敏感信息扫描 | 可写入 API Key 等密钥 | 安全隐患 |
| **R8** | PR 只看最新 Commit | 不注入完整 diff | Agent 缺乏全局视角 |
| **R9** | 无原子提交约束 | 可一次提交大量文件 | 提交难以审查和回滚 |

## 五、高级主题问题 (4项)

| ID | 问题 | 当前状态 | 影响 |
|:--:|:-----|:---------|:-----|
| **A1** | 无多阶段验证 | 依赖 LSP 和 subagent 汇报 | 缺乏语义级别的审查 |
| **A2** | 验证失败无自动回溯 | 仅提醒用户 | 需手动调整计划 |
| **A4** | 压缩丢失知识 | 仅保存 summary | 已解决的 Bug 模式丢失 |
| **A5** | 上下文顺序随机 | 无固定注入顺序 | Prompt Cache 命中率低 |
| **A6** | Verbosity 固定 | 不随 Token 余量调整 | 接近限制时仍输出冗长内容 |

---

## 六、测试验证问题 (5项)

| ID | 问题 | 当前状态 | 影响 |
|:--:|:-----|:---------|:-----|
| **T1** | 测试结果分散 | 依赖 `bun test` 分别执行 | 难以获得整体测试状态 |
| **T2** | TDD 拦截无帮助 | 仅返回错误信息 | 用户不知道如何写测试 |
| **T3** | 无 TDD 状态显示 | 不知道当前在 RED/GREEN | 难以跟踪 TDD 进度 |
| **T4** | 无会话质量评估 | 无法量化工作质量 | 不知道哪些会话需要改进 |
| **T5** | 无跨 Agent 验证 | 仅提醒用户验证 | 缺乏独立的质量检查 |

---

## 七、命令系统问题 (4项)

| ID | 问题 | 当前状态 | 影响 |
|:--:|:-----|:---------|:-----|
| **D1** | Prometheus 无强制风险矩阵 | 计划输出可能跳过风险评估 | 执行后才发现问题 |
| **D2** | /refactor 无清理功能 | 仅改写代码 | 遗留未使用的代码 |
| **D4** | 无场景化预设 | 每次手动配置 | 常用场景需重复配置 |
| **D6** | 无快捷清理命令 | 需手动配置 refactor | 死代码清理繁琐 |

---

## 八、代理系统问题 (4项)

| ID | 问题 | 当前状态 | 影响 |
|:--:|:-----|:---------|:-----|
| **G1** | Oracle 输出无结构 | 自由格式输出 | 难以提取工作量估算 |
| **G2** | Librarian 直接搜索 | 不先分析文档结构 | 可能错过重要内容 |
| **G3** | Explore 仅列文件 | 无优先级排序 | 关键文件埋没在列表中 |
| **G4** | Agent 交接信息丢失 | 依赖隐式 session_id | 下游 Agent 缺乏上下文 |
| **G5** | 无预定义协作链 | 每次临时组织 | Bugfix 和 Refactor 流程不一致 |

---

## 九、钩子系统问题 (4项)

| ID | 问题 | 当前状态 | 影响 |
|:--:|:-----|:---------|:-----|
| **H1** | 匹配逻辑硬编码 | 写死在 TS 中 | 无法通过配置调整匹配规则 |
| **H2** | 匹配不考虑环境 | 仅基于工具名 | 无法按包管理器等条件启用/禁用 |
| **H3** | 检查实时运行 | 每次工具调用都检查 | 高成本检查拖慢开发 |
| **H4** | 钩子失败影响主流程 | 无降级机制 | 一个钩子出错整个系统受影响 |

---

## Proposed Solution

从 everything-claude-code 的 `.qoder/repowiki/zh/` 目录复制已验证的最佳实践，适配到 oh-my-opencode。

- **Key approach**: 复制粘贴 + 适配（非从零开发）
- **Scope**: 仅增强现有功能，不新增架构组件
- **Estimated effort**: Large（43项，预计3-4周）

---

## Success Criteria

- [ ] 所有 43 项增强完成
- [ ] 每项增强有对应的测试用例
- [ ] `bun run typecheck` 0 错误
- [ ] `bun test` 全部通过
- [ ] `bun run build` 构建成功
- [ ] 现有功能无回归

## Risk Assessment

| Risk | Likelihood | Impact | Mitigation |
|------|------------|--------|------------|
| 破坏现有 API | Medium | High | 每项增强独立提交，便于回滚 |
| TDD 真实执行拖慢流程 | Low | Medium | 添加超时机制和开关 |
| 技能自动注入过度 | Medium | Medium | 提供配置项禁用 |
| 敏感信息扫描误报 | Medium | Low | 提供白名单配置 |
