# Oh My OpenCode

Claude Code, Codex, rastgele OSS modelleri arasında boğuşuyorsunuz. İş akışlarını yapılandırıyor, ajanları hata ayıklıyorsunuz.

Biz o işi yaptık. Her şeyi test ettik. Gerçekten sonuç verenleri sakladık.

OmO'yu kurun. `ultrawork` yazın. Tamamdır.


## Kurulum

### İnsanlar İçin

Bu istemi kopyalayıp LLM ajanınıza (Claude Code, AmpCode, Cursor vb.) yapıştırın:

```
Install and configure oh-my-opencode by following the instructions here:
https://raw.githubusercontent.com/code-yeongyu/oh-my-opencode/refs/heads/dev/docs/guide/installation.md

Veya [Kurulum Kılavuzu'nu](docs/guide/installation.md) okuyun, ama cidden, bırakın bir ajan yapsın. İnsanlar yapılandırma yaparken hata yapabilir.

```

### Ajan Orkestrasyonu

Sisyphus bir alt ajana delege ettiğinde model seçmez. Bir **kategori** seçer. Kategori otomatik olarak doğru modele eşlenir:

| Kategori             | Ne için kullanılır                    |
| :------------------- | :------------------------------------ |
| `visual-engineering` | Frontend, UI/UX, tasarım              |
| `deep`               | Otonom araştırma + yürütme            |
| `quick`              | Tek dosyalı değişiklikler, yazım hatası |
| `ultrabrain`         | Zor mantık, mimari kararlar           |

Ajan ne tür bir iş olduğunu söyler. Donanım (harness) doğru modeli seçer. Siz hiçbir şeye dokunmazsınız.

### Claude Code Uyumluluğu

Claude Code kurulumunuzu yaptınız. Güzel.

Her kanca (hook), komut, yetenek, MCP, eklenti burada değişmeden çalışır. Eklentiler dahil tam uyumluluk.

### Ajanlarınız İçin Dünya Klasmanında Araçlar

LSP, AST-Grep, Tmux, MCP sadece birbirine iliştirilmemiş, gerçekten entegre edilmiştir.

- **LSP**: `lsp_rename`, `lsp_goto_definition`, `lsp_find_references`, `lsp_diagnostics`. Her ajan için IDE hassasiyeti
- **AST-Grep**: 25 dilde desen uyumlu kod arama ve yeniden yazma
- **Tmux**: Tam etkileşimli terminal. REPL'ler, hata ayıklayıcılar, TUI uygulamaları. Ajanınız oturumda kalır
- **MCP**: Web araması, resmi dokümanlar, GitHub kod araması. Hepsi içinde

### Yeteneklere Gömülü MCP'ler

MCP sunucuları bağlam bütçenizi yer bitirir. Bunu düzelttik.

Yetenekler kendi MCP sunucularını getirir. İstek üzerine çalışır, göreve özeldir, iş bitince gider. Bağlam penceresi temiz kalır.

### Daha İyi Kod Yazar. Hash-Anchored Düzenlemeler

Donanım (harness) problemi gerçektir. Çoğu ajan hatası modelden kaynaklanmaz. Düzenleme aracından kaynaklanır.

> *\"Bu araçların hiçbiri modele değiştirmek istediği satırlar için kararlı, doğrulanabilir bir tanımlayıcı sunmaz... Hepsi modelin zaten gördüğü içeriği yeniden üretmesine dayanır. Başaramadığında -ki sık sık başaramazlar- kullanıcı modeli suçlar.\"*
>
> <br/>- [Can Bölük, The Harness Problem](https://blog.can.ac/2026/02/12/the-harness-problem/)

[oh-my-pi](https://github.com/can1357/oh-my-pi)'den esinlenerek **Hashline**'ı uyguladık. Ajanın okuduğu her satır bir içerik karması (hash) ile etiketlenmiş olarak döner:

```
11#VK| function hello() {
22#XJ|   return "world";
33#MB| }
```

Ajan bu etiketlere atıfta bulunarak düzenleme yapar. Eğer dosya son okumadan beri değiştiyse, karma eşleşmeyecek ve düzenleme bozulmadan önce reddedilecektir. Boşlukları yeniden üretmeye gerek yok. Güncelliğini yitirmiş satır hatalarına son.

Grok Code Fast 1: **%6.7 → %68.3** başarı oranı. Sadece düzenleme aracını değiştirerek.

### Derin Başlatma (Deep Initialization). `/init-deep`

`/init-deep` komutunu çalıştırın. Hiyerarşik `AGENTS.md` dosyaları oluşturur:

```
project/
├── AGENTS.md              ← proje genelindeki bağlam
├── src/
│   ├── AGENTS.md          ← src'ye özel bağlam
│   └── components/
│       └── AGENTS.md      ← bileşene özel bağlam
```

Ajanlar ilgili bağlamı otomatik olarak okur. Sıfır manuel yönetim.

### Planlama. Prometheus

Karmaşık bir görev mi var? Sadece istem yazıp dua etmeyin.

`/start-work` Prometheus'u çağırır. **Sizinle gerçek bir mühendis gibi mülakat yapar**, kapsamı ve belirsizlikleri belirler, koda dokunmadan önce doğrulanmış bir plan oluşturur. Ajan işe başlamadan önce ne inşa ettiğini bilir.

### Yetenekler

Yetenekler sadece istemlerden ibaret değildir. Her biri şunları getirir:

- Alana göre ayarlanmış sistem talimatları

Yerleşikler: `playwright` (tarayıcı otomasyonu), `git-master` (atomik commit'ler, rebase operasyonları), `frontend-ui-ux` (önce tasarım UI).

## Kaldırma

oh-my-opencode'u kaldırmak için:

1. **Eklentiyi OpenCode yapılandırmanızdan kaldırın**

   `~/.config/opencode/opencode.json` (veya `opencode.jsonc`) dosyasını düzenleyin ve `plugin` dizisinden `\"oh-my-opencode\"` ifadesini kaldırın:

   ```bash
   # jq kullanarak
   jq '.plugin = [.plugin[] | select(. != \"oh-my-opencode\")]' \\\
       ~/.config/opencode/opencode.json > /tmp/oc.json && \\\
       mv /tmp/oc.json ~/.config/opencode/opencode.json
   ```

2. **Yapılandırma dosyalarını kaldırın (isteğe bağlı)**

   ```bash
   # Kullanıcı yapılandırmasını kaldır
   rm -f ~/.config/opencode/oh-my-opencode.json ~/.config/opencode/oh-my-opencode.jsonc

   # Proje yapılandırmasını kaldır (varsa)
   rm -f .opencode/oh-my-opencode.json .opencode/oh-my-opencode.jsonc
   ```

3. **Kaldırma işlemini doğrulayın**

   ```bash
   opencode --version
   # Eklenti artık yüklenmiyor olmalı
   ```

- **Yapılandırma Konumları**: `.opencode/oh-my-opencode.jsonc` veya `.opencode/oh-my-opencode.json` (proje), `~/.config/opencode/oh-my-opencode.jsonc` or `~/.config/opencode/oh-my-opencode.json` (kullanıcı)
- **JSONC Desteği**: Yorumlar ve sondaki virgüller desteklenir
- **Ajanlar**: Herhangi bir ajan için modelleri, sıcaklıkları, istemleri ve izinleri geçersiz kılın
- **Yerleşik Yetenekler**: `playwright` (tarayıcı otomasyonu), `git-master` (atomik commit'ler)
- **Sisyphus Ajanı**: Prometheus (Planlayıcı) ve Metis (Plan Danışmanı) ile ana orkestratör
- **Arka Plan Görevleri**: Sağlayıcı/model başına eşzamanlılık sınırlarını yapılandırın
- **Kategoriler**: Alana özgü görev delegasyonu (`visual`, `business-logic`, özel)
- **Kancalar**: 25'ten fazla yerleşik kanca, hepsi `disabled_hooks` aracılığıyla yapılandırılabilir
- **MCP'ler**: Yerleşik web araması (Exa), dokümanlar (context7), GitHub araması (grep_app)
- **LSP**: Refaktör araçlarıyla tam LSP desteği
- **Deneysel**: Agresif kırpma, otomatik devam etme ve daha fazlası
